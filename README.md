# ⚾ 숫자 야구 게임 (Python 3.11+ 버전)

## 📌 환경 설정 및 요구사항
* **Python 버전**: Python 3.11 이상
* **의존성 관리**: `requirements.txt` 또는 `Poetry`, `uv` 중 택 1
* **테스트 도구**: `pytest`를 사용하여 핵심 도메인 로직에 대한 단위 테스트(Unit Test) 작성
* **코드 품질**: `black` 또는 `ruff`를 통한 포매팅, `mypy`를 통한 정적 타입 검사(Type Hinting 적극 활용)

## 🎮 진행 방법
* 숫자 야구 게임의 요구사항을 파악하고 MVC 패턴에 맞춰 설계한다.
* 도메인 로직과 UI 로직을 분리하여 객체지향적으로 구현한다.

## 🎯 기능 목록
### 랜덤 숫자 생성
* 랜덤 숫자는 3개로 구성된다. (`random` 모듈 활용)
* 랜덤 숫자는 서로 중복되지 않는다. **(중요)**
* 각 숫자는 1 ~ 9 사이의 정수이다.

### 사용자 입력받기
* 플레이어로부터 3개의 숫자를 입력받는다. (`input()` 함수 활용)
* 게임 종료 후 재시작/종료 여부(예: 1=재시작, 2=종료)를 입력받는다.

### 결과 판정 로직
* **스트라이크**: 같은 수가 같은 자리에 있으면
* **볼**: 같은 수가 다른 자리에 있으면
* **낫싱**: 같은 수가 전혀 없으면

### 게임 흐름 제어
* 정답(3스트라이크)을 맞출 때까지 입력을 반복한다.
* 게임 종료 시 게임을 다시 시작할지 완전히 끝낼지 선택한다.

## 🚨 예외 처리 목록
사용자가 잘못된 값을 입력할 경우 `ValueError`를 상속받은 커스텀 예외(예: `InvalidInputError`)를 발생시키고, 에러 메시지를 출력 후 다시 입력받도록 처리하거나 프로그램이 종료되도록 한다.
* 입력값이 없는 경우 (공백)
* 숫자가 아닌 값을 입력한 경우
* 입력 숫자가 3개 초과 혹은 미만인 경우
* 입력 숫자의 범위가 1 ~ 9가 아닌 경우 (예: 0이 포함된 경우)
* 입력 숫자에 중복이 있는 경우 (종류가 3개 미만인 경우)

## 🏗 프로그램 구조 : MVC 패턴

### 1. Controllers (`controllers.py`)
* **`Application` (진입점)**: `main.py`에 위치하며 전체 게임의 라이프사이클(시작, 재시작, 완전 종료)을 관리한다.
* **`GameController`**: 실제 한 판의 게임 동작 방식을 결정하고, 최대한 상태를 가지지 않게(stateless) 설계하여 Model과 View를 연결한다.

### 2. Views (`views.py`)
플레이어의 게임 화면(UI)을 담당한다. `print()`와 `input()`은 이 계층에서만 사용된다.
* **`InputView`**: 사용자의 입력을 받는 역할을 담당한다. (예: `read_numbers()`, `read_retry_command()`)
* **`OutputView`**: 게임의 결과, 에러 메시지, 진행 상황 등을 출력한다.

### 3. Models (`models.py`)
도메인 비즈니스 로직과 상태를 담당한다. (파이썬의 `@dataclass` 적극 활용 권장)
* **`Computer` / `AnswerGenerator`**: 정답 숫자를 생성하고 보관한다.
* **`Player`**: 사용자가 입력한 숫자 데이터를 검증하고 포장(Wrapping)한다.
* **`Referee`**: 플레이어의 숫자와 컴퓨터의 정답을 비교하여 결과를 도출한다.
* **`GameResult`**: 스트라이크, 볼 등의 비교 결과를 담는 데이터 구조이다. (`enum.Enum` 활용 권장)

### 4. Utils / Validators (`utils.py`, `validators.py`)
* **`validators.py`**: 입력값의 검증 로직을 분리하여 View나 Model에서 호출해 사용한다.
* **`utils.py`**: 공통으로 사용되는 함수를 모아둔다. 파이썬은 모듈 자체가 싱글톤으로 동작하므로 전역 함수나 모듈 레벨 변수로 관리하는 방식을 지향한다.
